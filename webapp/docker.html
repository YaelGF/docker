¿Qué es un Commit?

Puede ser útil confirmar los cambios o la configuración de un contenedor en una nueva imagen. Esto le permite depurar un contenedor ejecutando un shell interactivo o exportar un conjunto de datos de trabajo a otro servidor. Generalmente, es mejor usar Dockerfiles para administrar sus imágenes de una manera documentada y mantenible. Después de realizar algunos cambios dentro de un contenedor, decide que le gustaría conservar esos cambios. No desea perder esos cambios después de salir o detener el contenedor y le gustaría reutilizar este tipo de contenedor como base para otros.
